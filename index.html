Indicators

指標與評語指南

風險提醒不代表缺點！

它代表的是：最有可能影響你發表成功的因素。 請優先針對這些問題進行確認與排除。

資料源未確認 / 權限風險拿不到資料，Agent 就跑不動。
需外部系統整合技術門檻與時間成本通常較高。
題目範圍過大想做太多，容易導致全部都只做一半。
流程定義不夠清楚缺乏邏輯步驟，AI 無法照指示執行。

本次評審最重要的部分

很多題目不是做不到，而是：同時想做太多功能。

常見的過大規劃：
  • 試圖自動分析「所有」數據
  • 自動發送多方通知與信件
  • 產生極度完美的最終報告
  • 嘗試自動寫回原系統(ERP/CRM)
我們更希望大家：
先完成 MVP！

先完成一個真的能使用的單一流程版本。成功的 MVP (最小可行性產品)，通常比一個完成 20% 的巨型計畫更有價值。

Action Plan

接下來要做什麼？

請依照以下 5 個步驟，穩紮穩打推進您的 Agent。

1

閱讀專屬評分結果

仔細查看收到的評分結果，深入理解目前 Agent 的優勢與潛在風險在哪裡。

2

檢視 Scope 建議

盤點所有想做的功能，依照評語確認「哪些保留在第一版」、「哪些延後開發」。

3

關鍵步驟：定義 MVP

「如果發表日只能展示一個功能，你最希望展示什麼？」

能精準回答上面這個問題，通常那就是你現階段該專注全力開發的 MVP。

4

開始動手建置 Agent

不要等到所有問題都想通才開始做。動手實作，盲點與解法會在過程中自然浮現。

5

與 CoE 團隊討論

如果在任何階段對評分結果或技術實作有疑問，歡迎隨時找 CoE 團隊聊聊！

期待在 Demo Day 看到各位的 Agent 真正解決工作痛點！
