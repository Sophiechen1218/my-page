Agent 評分標準
完整說明

評分不是為了淘汰，而是為了確認一件事：
你的 Agent 真的解決了工作問題，而不只是展示品。

Level 2 評分的核心問題只有一個

這個 Agent 做出來了嗎？有真的在用嗎？讓工作變好了嗎？ 所有評分構面都圍繞這三個問題展開，而不是評估你的想法有多好——那是 Level 1 的事。

Why We Score This Way

評分原理：一個好的企業 Agent 該具備什麼？

評分標準不是憑感覺設計的，而是來自業界對「企業級 AI Agent」的核心要求。

根據 MIT Sloan Management Review、Gartner 及 McKinsey 對企業 AI 導入的研究，一個真正為企業創造價值的 AI Agent，不在於技術有多複雜，而在於是否滿足以下條件： ① 輸出可信賴且穩定、② 有真實使用紀錄佐證、③ 讓流程產生可量化的改變、④ 能被團隊共用而非個人獨佔、⑤ 知識可沉澱傳承不因人員異動而消失。 本次評分標準即以此五項能力為基礎設計。

輸出可信賴

準確、穩定、可直接使用，不需大量人工後處理

真實使用佐證

有紀錄可證明 Agent 已融入日常工作，而非展示品

流程可量化改善

有具體數字說明改善了什麼，而非「感覺比較快」

團隊共用性

不只作者會用，同仁也能獨立上手操作

知識可傳承

設計邏輯有文件化，不因人員異動而消失

兩階段的評審視角差異

複賽｜CoE 主管視角

關注執行品質與落地細節——Agent 做出來好不好、跑得穩不穩、有沒有真的在用、別人能不能接手。

決賽｜老闆視角

關注商業價值與組織影響力——這個 Agent 值不值得投資、有沒有讓團隊受益、能不能為組織留下知識資產。

Semi-Final Scoring

複賽評分標準 AI 評分輔助

CoE 將使用 AI 工具輔助評分，依據以下五項構面對每份繳交的簡報與 Agent 進行系統性評估，確保評分一致性。

AI 評分說明

CoE 將建立結構化評分 Agent，依下列五項構面的評分標準，對每份繳交內容進行初步評分並產出評分報告，再由 CoE 人工複核確認。此機制確保 90 份以上的繳交內容能在一致標準下被公平評估。

評分構面 · 共 5 項

每項 20 分，滿分 100 分

評分構面 CoE 審查重點 給分參考 配分
1輸出準確性
這個輸出能直接用嗎？
 CoE 依 Agent Card 使用情境輸入測試案例，觀察實際輸出
· 回應內容是否正確且完整，無明顯錯誤
· 輸出格式是否符合實際工作使用需求
· 輸出是否可直接使用，或需大量後處理
 18–20：準確完整，可直接使用
11–17：大致正確，需少量修改
0–10：錯誤明顯或需大量後處理 		20 分
2執行穩定性
每次跑都一樣嗎？
 CoE 重複操作 Agent 並比對繳交之驗證紀錄（至少 3 次）
· 相同輸入多次執行，核心輸出是否一致
· 非預期輸入時是否有合理的回應或邊界提示
· 驗證紀錄是否呈現不同情境皆可正常運作
 18–20：多次執行高度一致，邊界處理佳
11–17：大致穩定，偶有差異但可接受
0–10：結果飄移明顯或驗證不足 		20 分
3真實使用紀錄
這是平常在用的嗎？
 簡報或附件中須呈現來自真實工作的使用紀錄
· 對話紀錄或截圖是否來自真實工作情境（非刻意設計）
· 是否呈現使用頻率或使用次數
· 輸出是否已實際用於工作產出
 18–20：有明確使用紀錄，持續在用中
11–17：少量紀錄或仍在測試階段
0–10：僅展示案例，無實際使用證明 		20 分
4流程改善的具體性
Before/After 的差距是真的嗎？
 不評估「是否有改善潛力」（Level 1 已評），改評「實際改善了多少」
· 哪些人工步驟被 Agent 取代或簡化（需明確列出）
· 時間節省或品質提升是否有具體數字
· 數字是否有可對比的基準，非主觀描述
 18–20：改善步驟明確，數字具體可信
11–17：有改善但數字薄弱或估算模糊
0–10：改善不明確或沿用 Level 1 規劃內容 		20 分
5團隊可移交性
這個人不在，還能用嗎？
 CoE 以「第一次使用者」視角操作，不看任何額外說明
· 命名與功能描述是否讓陌生使用者看得懂
· 是否有使用提示或限制說明內建在 Agent 中
· 不具備技術背景的同仁是否也能正確操作
 18–20：陌生使用者可完全自行上手
11–17：需簡短說明才能使用
0–10：高度依賴原作者，無法獨立使用 		20 分
複賽 Agent 總分 100 分

※ 部門平均分數＝部門 Agent 總得分 ÷ 部門人數，用於部門獎金計算。AIS Team 分台北／高雄；Marketing+ESD+UX、Presales+Tiger+CN Tech、Finance+HR 各視為同一部門。

Final Scoring

決賽評分標準

高層評審依預錄影片與簡報，從老闆視角評估 Agent 的商業價值與組織影響力。

評分構面 · 共 5 項

每項 20 分，滿分 100 分

評分構面 高層評審重點 給分參考 配分
1商業價值清晰度
這幫公司省了什麼？
 影片中是否以非技術語言清楚陳述 Agent 對公司的貢獻
· 價值陳述是否簡潔有力，不需背景知識即可理解
· 是否以部門或公司層級說明貢獻（非純個人視角）
· 數字是否讓人一聽就有感（如：省下每週 X 小時）
 18–20：價值一目瞭然，數字有感
11–17：方向正確但陳述需要解釋
0–10：價值不明確或充斥技術術語 		20 分
2真實場景說服力
這是真的在跑，還是今天才用？
 影片是否讓評審感受到 Agent 已深度融入日常工作
· 影片輸入是否來自真實工作內容（非虛構範例）
· 輸出是否直接對應真實工作產出
· 影片呈現的使用方式是否自然，而非照稿演示
 18–20：場景完全真實，融入日常
11–17：部分真實，仍有展示痕跡
0–10：明顯為發表設計，缺乏真實感 		20 分
3可量化的改善成果
這個投入值得嗎？
 複賽已審數字合理性，決賽進一步評估數字的說服力與可信度
· 數字是否有明確來源與計算邏輯
· 改善是否持續發生，而非一次性效益
· 是否能讓高層判斷「值不值得持續投入」
 18–20：數字有據，持續改善，具說服力
11–17：數字合理但深度稍不足
0–10：數字模糊或僅重複 Level 1 預估 		20 分
4組織影響力
只有一個人在用嗎？
 影片或簡報中是否呈現 Agent 對團隊或部門層級的實際影響
· 是否有其他成員也在使用（自然採用，非被要求）
· 是否解決了整個部門的共同痛點，而非純個人需求
· 影響範圍是否正在擴大，有擴散趨勢
 18–20：團隊已受益，自然擴散中
11–17：部分成員受益，仍以個人為主
0–10：影響範圍僅限個人 		20 分
5知識沉澱與傳承性
這個知識會因人員異動消失嗎？
 不要求跨部門複製，但要求使用邏輯與操作方式有完整文件化
· 是否有說明 Agent 的設計思路與使用限制
· 操作方式是否有文件記錄，新人也能接手
· 這個 Agent 是否為部門留下可傳承的知識資產
 18–20：設計邏輯與操作完整文件化
11–17：有部分說明但不夠完整
0–10：無文件，知識僅存於作者腦中 		20 分
決賽 Agent 總分 100 分

※ 個人獎金依決賽總分排序取前三名。評審建議：CEO / C-level 1 名 ＋ 主管代表 2 名。
